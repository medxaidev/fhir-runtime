## Overview

v0.9.0 completes **STAGE-B: fhir-server Prerequisites**, resolving three gaps identified in the architecture review (GAP-01, GAP-02, GAP-03) that were blocking `fhir-server` v0.1.0 development.

**Key Achievements**:
- **B1**: `RemoteTerminologyProvider` interface for delegating terminology operations to external servers
- **B2**: `validateMany()` batch validation API for transaction Bundle processing
- **B3**: `SnapshotCache` with lazy generation, concurrent deduplication, and server-side warmup

---

## What's New

### B1: RemoteTerminologyProvider Interface

**Source**: `src/provider/remote-terminology-provider.ts`  
**Exports**: 7 types (1 interface + 6 param/result types)

Defines the contract for delegating `$expand`, `$validate-code`, and `$lookup` operations to remote terminology servers. fhir-runtime does NOT implement HTTP calls ã this interface is designed to be implemented by `fhir-server`.

```typescript
interface RemoteTerminologyProvider {
  expandValueSet(params: RemoteExpandParams): Promise<unknown>;
  validateCode(params: RemoteValidateCodeParams): Promise<RemoteValidateCodeResult>;
  lookupCode(params: RemoteLookupParams): Promise<RemoteLookupResult>;
}
```

**Runtime injection**:
```typescript
const runtime = await createRuntime();
runtime.setRemoteTerminologyProvider(myHttpProvider);
// Later:
const provider = runtime.getRemoteTerminologyProvider();
```

### B2: Batch Validation API

**Added to**: `FhirRuntimeInstance`  
**Exports**: 2 types (`BatchValidationOptions`, `BatchValidationResult`)

Supports concurrent validation of multiple resources with configurable concurrency and fail-fast mode.

```typescript
const result = await runtime.validateMany([
  { resource: patient1, profileUrl: 'http://hl7.org/fhir/StructureDefinition/Patient' },
  { resource: patient2, profileUrl: 'http://hl7.org/fhir/StructureDefinition/Patient' },
  { resource: obs1, profileUrl: 'http://hl7.org/fhir/StructureDefinition/Observation' },
], { concurrency: 4, failFast: false });

console.log(result.valid);      // true/false
console.log(result.errorCount); // total errors across all resources
console.log(result.results);    // per-resource ValidationResult[]
```

### B3: SnapshotCache ã Lazy Snapshot Generation

**Source**: `src/definition/snapshot-cache.ts`  
**Exports**: 1 class (`SnapshotCache`)

- On-demand snapshot generation with LRU-style caching
- Concurrent deduplication: multiple requests for the same SD share one generation promise
- Server-side warmup API for pre-loading common resource types

```typescript
// Option: snapshotMode (default: 'eager')
const runtime = await createRuntime({ snapshotMode: 'lazy' });

// Pre-warm common types after startup
await runtime.warmupSnapshots(['Patient', 'Observation', 'Encounter']);

// Monitor cache
console.log(runtime.getSnapshotCacheSize()); // 3
```

## fhir-server Integration Path

After v0.9.0, `fhir-server` can:

```typescript
import { createRuntime } from 'fhir-runtime';
import type { RemoteTerminologyProvider } from 'fhir-runtime';

// 1. Create runtime with lazy snapshots
const runtime = await createRuntime({ snapshotMode: 'lazy' });

// 2. Inject HTTP-based terminology provider
runtime.setRemoteTerminologyProvider(new HttpRemoteTerminologyProvider(client));

// 3. Warm up common types
await runtime.warmupSnapshots(['Patient', 'Observation', 'Encounter', 'Condition']);

// 4. Validate transaction Bundle
const result = await runtime.validateMany(bundleEntries, { failFast: true });
```
